City Without Crime

"Project done with hp."

PROJECT GOALS Implementing a digital criminal database.

This was my summer project in 2013 done while my training in hp. This project is aimed at creating a web application that focuses on lodging complains by victims. The systems will maintain a secure data base where data can stored or fetched from. These systems enhance the ability of law professionals to coordinate care by providing a victimâ€™s case information and visit history at the place and time that it is needed.

TECHNOLOGY STACK Tools to bring the project to life

This website is an integrated crime management system that is deployed on a 2 tier architecture methodology. This includes a GUI where the users are provided with there respective activities and a physical layer consisting of the database.The basic designing is done by HTML, CSS and Javascript whereas the backend is a combination of PHP-Mysql.

PLANNING Functional and non functional requirements

The functional requirements included points like First Information Report, Missing Person Report, Anonymous Report, Check case status, Admin Login.

While the non-functional requirements included points like Minimum Speed of 144Kbps, Backend password protected, Compatible with any O.S with internet connectivity, New modules can be easily incorporated.

OVERVIEW A brief about the project

According to the function and non-functional requirements mentioned earlier, we proceeded with the project. We were a team of five and as the team leader, i divided the workfore for the swift completion of the project. This project was capable of storing information provided by the victims, according to the forms provided by the website, so that it can be accessed by the police officers on the administrater's side.
